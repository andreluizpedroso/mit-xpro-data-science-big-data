Introdução

O objetivo deste estudo é experimentar vários modelos de recomendação para prever as preferências de filmes dos usuários, utilizando um banco de dados de uso livre. Ao concluir, você terá adquirido experiência prática para:

  • Carregar um conjunto de dados grande
  • Criar vários modelos
  • Avaliar as previsões desses modelos
  • Utilizar ferramentas como a validação cruzada para treinar e testar modelos
  • Entender parâmetros como RMSE, precisão e exatidão @k e top n recomendações

Este documento explicará como trabalhar com esses dados e a estrutura para obter resultados significativos. Não esqueça de salvar seu notebook frequentemente para não perder seu trabalho!

Configuração do notebook

Um dos primeiros passos em qualquer tarefa de ciência de dados é importar as ferramentas necessárias que serão utilizadas. Veja a seção “Importar” do notebook e siga as instruções para importar as bibliotecas necessárias. Além das bibliotecas mais comuns em ciência de dados, como pandas ou matplotlib, este estudo de caso usará a biblioteca [surprise] (http://surpriselib.com/) para gerar os diferentes modelos de recomendação.

Dados

Observe a seção “Dados” do notebook e siga as instruções para carregar o banco de dados MovieLens, especificamente, a versão ml-100k, que inclui 100.000 pontuações de 1700 filmes.

Validação cruzada

A validação cruzada será amplamente usada neste estudo de caso para treinar e avaliar os modelos. Essa estratégia é baseada na ideia de particionar o conjunto de dados em dois subconjuntos de treinamento/teste, com a qual você deve estar familiarizado.

No entanto, em vez de fazer uma única separação de dados, faremos várias. Cada separação de dados é chamada de “iteração”. Neste estudo de caso, k denota o número de iterações que são usadas; k = 5 é um valor comumente usado.

Esta imagem fornece uma explicação visual de como funciona a validação cruzada.

Modelos de recomendação

Modelo 1: Aleatório

Primeiramente, devemos obter um ponto de referência para nosso modelo. E a melhor maneira fazer isso é com um algoritmo aleatório! Em essência, esse primeiro algoritmo não está personalizado com as preferências de nenhum usuário; a pontuação de filmes só é atribuída às preferências a partir da distribuição inicial dos dados.

Vá para a seção “Modelo 1: Aleatório” do seu notebook e siga as instruções para criar um novo modelo, treiná-lo com os dados e avaliar seu RMSE.

Modelo 2: Filtragem colaborativa baseada no usuário

É claro que podemos obter algo muito melhor do que uma suposição aleatória de pontuação de filmes! O modelo a seguir usará a noção de similaridade definida pelos usuários para aplicar a filtragem colaborativa (user-user collaborative filtering).

Vá para a seção “Modelo 2: Filtragem colaborativa baseada no usuário” do seu notebook e siga as instruções para criar um novo modelo, treiná-lo com os dados e avaliar seu RMSE.

Modelo 3: Filtragem colaborativa baseada em itens

O modelo a seguir usará a noção de similaridade baseada em itens para mais uma vez aplicar a filtragem colaborativa (item-item collaborative filtering).

Vá para a seção “Modelo 3: Filtragem colaborativa baseada em itens” do seu notebook e siga as instruções para criar um novo modelo, treiná-lo com os dados e avaliar seu RMSE.

Modelo 4: Fatoração de matrizes

O modelo final neste estudo de caso usará a abordagem da fatoração de matriz com o algoritmo SVD (singular value decomposition) para tentar prever as pontuações dos filmes do usuário. Nesse estudo, tentaremos determinar uma estrutura matemática subjacente na matriz de pontuações do usuário para que ela possa ajudar a prever pontuações ausentes no futuro.

Vá para a seção “Modelo 4: fatoração de matriz” de seu notebook e siga as instruções para criar um novo modelo, treiná-lo com os dados e avaliar seu RMSE.

Avaliação dos modelos - Precisão e exatidão @k

O RMSE não é único parâmetro que pode ser utilizado aqui. Também podem ser examinadas duas métricas fundamentais nos sistemas de recomendação: precisão e exatidão (precision and recall). Além disso, acrescentamos um parâmetro kkk que ajuda na hora de entender problemas com múltiplas saídas de pontuação.

Vá para a seção “Precisão e exatidão @ kkk” do seu notebook e siga as instruções para criar um novo modelo, treiná-lo com os dados e avaliar seu RMSE.

Avaliação das Top n previsões

Finalmente, precisamos ver realmente as pontuações que o modelo prevê para os usuários. Pode-se variar o número de filmes nnn recomendados por cada usuário.

Vá para a seção “Top n previsões” do seu notebook e siga as instruções para calcular as previsões de pontuações para alguns usuários.
